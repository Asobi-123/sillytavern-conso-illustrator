# 悬浮面板手动测试清单

本清单只针对 `feat/floating-panel-ui` 引入的悬浮面板 UI。

## 前置条件

- 已执行 `npm run build`
- 扩展已加载到可运行的 SillyTavern 环境
- 至少打开一个聊天
- 生图功能本身可用

## 1. 面板外壳

- 打开悬浮面板
  - 预期：面板正常出现，无明显报错
- 关闭悬浮面板
  - 预期：面板隐藏，保留重新打开入口
- 再次打开悬浮面板
  - 预期：面板可正常恢复
- 刷新页面后检查开关状态
  - 预期：面板开关状态能保持

## 2. 主题切换

- 在主控台切换多套主题
  - 预期：面板颜色立即切换
- 分别测试一套深色和一套浅色
  - 预期：文字、输入框、overlay、子卡片都可读
- 打开 `上下文注入设置`
  - 预期：overlay 配色也会跟随主题
- 打开 `世界书管理`
  - 预期：世界书卡片和条目行也会跟随主题

## 3. 主控台

- 切换 `启用自动插画`
  - 预期：底层设置同步变化，并在刷新后保持
- 在主控台切换提示词生成模式
  - 预期：提示词设置页显示对应模式内容
- 修改当前聊天图片文件夹标签
  - 预期：当前聊天标签更新并在聊天切换后正确刷新

## 4. 提示词设置

### 共享 API 模式

- 切到共享 API 模式
  - 预期：只显示 `共享 API 模式` 区域
  - 预期：`独立 API 模式` 区域隐藏
- 切换元提示预设
  - 预期：预览同步变化
- 进入预设编辑
  - 预期：编辑器正常出现，保存按钮状态正常

### 独立 API 模式

- 切到独立 API 模式
  - 预期：只显示 `独立 API 模式` 区域
- 修改基础参数
  - 预期：数值同步更新并能持久化
- 切换 `上下文注入`
  - 预期：底层上下文相关 checkbox 同步
- 切换 `世界书注入`
  - 预期：底层世界书注入 checkbox 同步
- 折叠/展开 `指南预设`
  - 预期：独立工作，不影响其他块
- 折叠/展开 `独立 LLM API`
  - 预期：独立工作，不影响其他块

### 两种模式共用

- 打开 `角色固定 Tag`
  - 预期：overlay 打开，编辑器可用
- 编辑 `提示词检测与风格`
  - 预期：输入、重置等原有逻辑仍可用

## 5. 独立生图

- 打开 `独立生图` 页面
  - 预期：standalone 内容嵌入面板中
- 切换 `上下文注入`
  - 预期：只影响 standalone 的 include-character-info
- 切换 `世界书注入`
  - 预期：只影响 standalone 的 include-world-info
- 在 AI 模式下生成提示词
  - 预期：出现 prompt 卡片
- 使用长文本 `全屏` 按钮编辑 prompt
  - 预期：全屏编辑器打开，应用后写回原卡片
- 在手动 Prompt 模式下使用 `全屏`
  - 预期：全屏编辑器打开，应用后写回手动输入框
- 从 prompt 卡片生成图片
  - 预期：卡片里显示预览

## 6. 世界书管理

- 打开 `世界书管理`
  - 预期：overlay 正常出现
- 搜索世界书
  - 预期：列表过滤正常
- 折叠/展开某本世界书
  - 预期：条目列表按书名正常切换
- 启用/禁用单个条目
  - 预期：当前聊天的配置被更新并保留

## 7. 画廊

- 打开 `画廊` 页面
  - 预期：现有 gallery widget 被嵌入面板，而不是继续独立漂浮
- 展开/折叠消息分组
  - 预期：行为正常
- 从画廊打开图片查看器
  - 预期：原有图片查看器功能正常

## 8. 全屏文本编辑器

- 打开 `元提示` 的全屏编辑/预览
  - 预期：overlay 打开
- 打开 `频率指南` 的全屏编辑/预览
  - 预期：overlay 打开
- 打开 `提示词编写指南` 的全屏编辑/预览
  - 预期：overlay 打开
- 打开 `独立生图 prompt` 的全屏编辑
  - 预期：overlay 打开
- 对可编辑目标点击 `应用`
  - 预期：原字段更新
- 对只读预览目标打开全屏
  - 预期：只读显示，隐藏应用按钮

## 9. 回归检查

- 点击聊天里的 AI 生成图片
  - 预期：原有的图片操作弹窗仍然出现
  - 预期：悬浮面板不会接管或替换这条逻辑
- 切换聊天
  - 预期：面板仍稳定
  - 预期：当前聊天的世界书和文件夹标签随聊天刷新
- 在聊天流式生成时测试
  - 预期：原有进度提示和流式预览系统仍正常
